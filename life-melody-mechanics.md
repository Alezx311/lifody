## 1. Ядро симуляції — Мелодичне ДНК

### 1.1 Що таке ДНК клітини

Кожна жива клітина несе **мелодичний геном** — масив з N значень, де кожне значення кодує одну ноту:

```
ДНК = [ pitch, duration, velocity, articulation ][]
```

| Параметр | Діапазон | Що кодує |
|---|---|---|
| `pitch` | 0–11 (ступінь ладу) | Висота ноти |
| `duration` | 1–8 (тривалість у 16-х) | Тривалість ноти |
| `velocity` | 0–127 | Гучність / акцент |
| `articulation` | 0–3 | legato / staccato / tenuto / accent |

Початкова довжина геному — **4 ноти**. Геном може **зростати** до 16 нот через мутації та злиття.

---

### 1.2 Правила народження клітини (розширення правил Life)

У класичній грі Life клітина народжується якщо має рівно 3 живих сусіди. Тут додається крок **наслідування ДНК**:

```
Нова клітина = crossover(сусід_А, сусід_Б, сусід_В)
```

**Алгоритм crossover:**

1. Обрати двох найстарших сусідів як "домінантних батьків"
2. Для кожної позиції в геномі — взяти ноту від одного з батьків (50/50)
3. Якщо батьки мають різну довжину геному — взяти довжину старшого
4. Застосувати **мутацію** з ймовірністю `mutation_rate`

**Формула mutation_rate:**

```
mutation_rate = base_rate × temperature_modifier × (1 - fitness_score / 100)
```

Тобто клітини з **низьким фітнесом мутують частіше** — середовище тисне на них до змін.

---

### 1.3 Вік клітини та ДНК

Кожна клітина накопичує **вік** (кількість тіків що вона прожила):

- Вік 0–5: **ювенільна** — нота грає тихо, articulation = staccato
- Вік 6–20: **зріла** — нота грає на повну силу
- Вік 21+: **стара** — pitch поступово "дрейфує" на ±1 ступінь, додаючи вібрато

---

### 1.4 Кластери — живі організми

Група сусідніх живих клітин утворює **кластер**. Кластер — це основна одиниця взаємодії для гравця.

**Властивості кластера:**

| Властивість | Як обчислюється | Музичний ефект |
|---|---|---|
| **Розмір** | Кількість клітин | Гучність (більший = гучніший) |
| **Площа bbox** | Ширина × висота bounding box | Октава (компактний = нижче) |
| **Щільність** | розмір / площа bbox | Тембр (густий = темніший) |
| **Вік кластера** | Тік народження першої клітини | Атака (старий = плавніший) |
| **Fitness score** | Накопичені оцінки гравця | Шанс виживання клітин |
| **Мелодія** | Усереднений геном усіх клітин | Відтворювана послідовність нот |

**Мелодія кластера** будується так:
1. Зібрати всі геноми клітин кластера
2. Впорядкувати клітини зверху-вниз, зліва-направо
3. Прочитати їх ноти послідовно → отримати мелодичну фразу
4. Обрізати або зациклити до кратної такту довжини

## 2. Музичний фітнес — Природний відбір через слух

### 2.1 Fitness Score

Кожен кластер має `fitness_score` від 0 до 100 (початково 50).

**Зміна фітнесу:**

| Дія гравця | Зміна fitness |
|---|---|
| Натиснути "подобається" на кластер | +15 |
| Довго утримувати (заглушити) | −20 |
| Кластер існує N тіків без оцінки | −0.1 за тік (природне "старіння") |
| Кластер досяг стабільного стану | +5 автоматично |
| Мелодія кластера схожа на збережену улюблену (>70%) | +10 |
| Мелодія кластера частково схожа (50–70%) | +5 |

### 2.2 Як фітнес впливає на виживання

В стандартних правилах Life клітина **завжди** вмирає при < 2 або > 3 сусідах. Тут додається **модифікатор виживання**:

```
survival_chance = base_chance + (fitness_score - 50) × 0.005
```

Тобто кластер з fitness = 100 має на 25% вищий шанс виживання кожної клітини. Кластер з fitness = 0 — на 25% нижчий. Симуляція залишається впізнаваною Life, але не детермінованою.

### 2.3 Розповсюдження фітнесу

Коли кластер **ділиться** або **зливається**:

- Злиття: `новий_fitness = (fitness_A × size_A + fitness_B × size_B) / (size_A + size_B)`
- Поділ: обидва дочірні кластери наслідують fitness батька ± 5 (випадково)

### 2.4 Зберігання улюблених мелодій

Гравець може натиснути **"Зберегти"** на кластері — його мелодія записується у **Бібліотеку**. Далі:
- Система порівнює всі нові мелодії з бібліотекою через edit distance
- Схожі мелодії отримують автоматичний бонус фітнесу
- Бібліотека видима гравцю і відтворювана в будь-який момент

## 3. Інструменти впливу на середовище

Гравець має **11 інструментів** для впливу на середовище (клавіші 1–0 + додаткові через UI):

---

### 🌡️ Інструмент 1 — Теплові зони

Гравець малює зони двох типів пензлем довільного розміру.

**Гаряча зона** (червона):

| Параметр | Стандарт | У гарячій зоні |
|---|---|---|
| Народжується при сусідах | 3 | 2 або 3 |
| Виживає при сусідах | 2–3 | 1–3 |
| mutation_rate | base | base × 2.5 |
| Музичний ефект | — | Дисонанс, хроматика, швидкий темп |

**Холодна зона** (синя):

| Параметр | Стандарт | У холодній зоні |
|---|---|---|
| Народжується при сусідах | 3 | Тільки 3 |
| Виживає при сусідах | 2–3 | Тільки 3 |
| mutation_rate | base | base × 0.2 |
| Музичний ефект | — | Консонанс, мінімалізм, повільний темп |

**Теплова зона не постійна** — вона поступово розсіюється за 30 тіків якщо гравець не підтримує її.

---

### 🧲 Інструмент 2 — Атрактори та Репелери

Гравець ставить **маяки** на сітку. Маяк не є живою клітиною — це поле впливу.

**Атрактор** (зелений маяк):
- Глайдери та рухомі кластери у радіусі R відчувають "тягу"
- Реалізується через збільшення survival_chance клітин на стороні що ближча до маяка
- Ефект: притягує кластери → збільшує шанс злиття → гібридизація мелодій

**Репелер** (помаранчевий маяк):
- Протилежний ефект — відштовхує кластери
- Використання: ізолювати цінний кластер від небажаних зіткнень

**Резонансний маяк** (золотий):
- Особливий тип — шукає два кластери з найближчими мелодіями
- Підсилює їх взаємне притяжіння
- Ефект: сприяє злиттю "споріднених" мелодій

Одночасно на сітці може бути не більше **5 маяків** (обмеження ресурсів гравця).

---

### 💉 Інструмент 3 — Ін'єкція ДНК

Гравець **вводить мелодичний фрагмент** у симуляцію.

**Процес:**
1. Гравець відкриває Піаноролл-редактор (до 8 нот)
2. Малює або записує мелодичний фрагмент
3. Вибирає точку на межі сітки
4. Система породжує 1–3 клітини з цим геномом у вибраній точці
5. Далі — природній відбір: або вони виживуть і розповсюдять ДНК, або загинуть

**Обмеження:** ін'єкція доступна раз на **15 тіків** (cooldown).

**Ризик:** введений ДНК одразу ж починає мутувати. Якщо не підтримувати fitness таких клітин — оригінальна мелодія зміниться.

---

### ⏳ Інструмент 4 — Часовий відкат

Гравець може **перемотати один кластер** на N тіків назад.

**Механіка:**
- Система зберігає snapshot кожного кластера кожні 5 тіків (до 10 знімків)
- Гравець вибирає кластер → відкривається Timeline з мініатюрами станів
- Вибирає момент → кластер повертається до того стану, але **в поточній позиції сітки**
- Решта симуляції продовжується без змін

**Використання:**
- Мелодія кластера "зіпсувалася" через небажане злиття → відкотити до кращого стану
- Нестандартна тактика: відкотити кластер і перенаправити його через атрактор

**Обмеження:** 3 відкати всього за сесію. Витрачати треба обдумано.

---

### ✂️ Інструмент 5 — Розкол

Гравець **розрізає кластер** по лінії що він малює.

**Механіка:**
- Лінія розрізу визначає які клітини відходять до кластера A і B
- Кожен дочірній кластер отримує підмножину геномів батьківських клітин
- Мутація: 2–3 ноти на межі розрізу **замінюються** випадковими з поточного тонального регіону
- Fitness ділиться пропорційно розміру: більший дочірній отримує більше

**Навіщо:**
- Великий кластер з "нецікавою" мелодією можна розбити в надії що один з фрагментів еволюціонує краще
- Відокремити "хорошу" частину від "поганої" перед зіткненням з іншим кластером

---

### 🖌️ Інструмент 6 — Малювання клітин (DRAW)

Гравець може **вручну малювати живі клітини** на сітці (клавіша 9).

**Механіка:**
- Клітина створюється з випадковим геномом, обмеженим тональним регіоном у точці малювання
- Дозволяє швидко створювати початкові конфігурації або "засівати" порожні ділянки

---

### 🎨 Інструмент 7 — Малювання тональних зон (PAINT_REGION)

Доступний лише в режимі **Island Map**. Гравець малює тональні зони пензлем.

**Механіка:**
- Вибирається тоніка (0–11) та тип ладу
- Гравець "фарбує" клітини сітки, присвоюючи їм обраний тональний регіон
- Дозволяє створювати довільну географію ладів

---

## 4. Гармонічне середовище — Тональні регіони

### 4.1 Режими карти

Система тональних регіонів підтримує **два режими карти**:

**MAP_STANDARD** — конфігуровна сітка тональних зон з пресетами:

| Пресет | Сітка | Опис |
|--------|-------|------|
| Classic | 3×2 | Оригінальні 6 регіонів (До мажор, Соль мажор, Ре мажор, Ля мінор, Мі мінор, Фрігійський) |
| 12Maj | 4×3 | Усі 12 хроматичних тонік, мажорна гама |
| 12Min | 4×3 | Усі 12 тонік, натуральний мінор |
| 5ths | 4×3 | Квінтове коло, усі мажорні |
| 7Modes | 4×2 | 7 ладів До мажору + Хроматика |
| Dorian | 4×3 | Усі 12 тонік, дорійський лад |

**MAP_ISLAND** — 12 нотних островів (розташовані як клавіші піаніно: 7 білих + 5 чорних), вільне малювання тональних зон інструментом PAINT_REGION.

### 4.2 Доступні лади (11 типів)

Мажор, Натуральний мінор, Дорійський, Фрігійський, Лідійський, Міксолідійський, Локрійський, Гармонічний мінор, Пентатоніка мажорна, Пентатоніка мінорна, Блюз.

### 4.3 Класичний пресет (оригінальний)

```
┌──────────────┬──────────────┬──────────────┐
│   До мажор   │  Соль мажор  │  Ре мажор    │
│  (іонійський)│  (іонійський)│  (іонійський)│
├──────────────┼──────────────┼──────────────┤
│   Ля мінор   │  Мі мінор    │  Фрігійський │
│  (еолійський)│  (еолійський)│              │
└──────────────┴──────────────┴──────────────┘
```

### 4.4 Вплив регіону на клітини

- Клітина **народжується** в регіоні → її pitch обмежений нотами цього ладу
- Клітина **мігрує** через межу регіону → pitch поступово транспонується до нового ладу (1 ступінь за 3 тіки)
- Глайдер що перетинає кілька регіонів → мелодія **модулює** — ефект гармонічної подорожі

### 4.5 Межі між регіонами

Межі між регіонами — **не тверді стіни**, а **зони переходу** шириною 3 клітини. У цій зоні:
- Клітини можуть мати ноти з обох сусідніх ладів одночасно
- Mutation_rate підвищений × 1.5 — зони "творчого хаосу"
- Злиття кластерів з різних регіонів у цій зоні дає найцікавіші гібридні мелодії

## 5. Події-каталізатори

Одноразові драматичні дії з довгим cooldown. Гравець отримує одну подію кожні **50 тіків** (може накопичити до 3).

---

### ☄️ Метеорит

**Ефект:** Вбиває всі клітини у радіусі R.

**Але:** перед загибеллю кожна клітина "випускає спору" — окрему клітину з її геномом що з'являється на випадковій позиції за межами радіусу. Шанс появи спори: `fitness_score / 100`.

**Стратегічне використання:**
- Знищити великий "нецікавий" кластер і сподіватися що з спор виросте щось краще
- Звільнити місце для розвитку кращих кластерів

---

### 🤝 Резонанс

**Ефект:** Система знаходить два кластери з найбільш схожими мелодіями і телепортує їх ближче одне до одного (у межах 10 клітин).

**Музичний результат:** злиття схожих мелодій → поглиблення, розвиток спільної теми замість хаотичного схрещення.

---

### ❄️ Заморозка

**Ефект:** Гравець вибирає кластер і "кристалізує" його на 20 тіків.

**Заморожений кластер:**
- Не змінює форму (не підкоряється правилам Life)
- Не мутує
- Але продовжує **грати свою мелодію**
- Сусідні клітини що народжуються поруч — **наслідують його ДНК з ймовірністю 70%**

**Використання:** знайшли чудову мелодію → заморозити → дозволити їй "заразити" сусідів.

---

### 🌊 Хвиля мутацій

**Ефект:** По всій сітці прокочується хвиля підвищеної мутації (як Гаряча зона але тимчасова і тотальна).

- Тривалість: 10 тіків
- Mutation_rate × 5 для всіх клітин
- Fitness scores всіх кластерів скидаються на 50 (tabula rasa)

**Використання:** якщо симуляція "застрягла" у нецікавому стані → радикально скинути і дати нову точку відліку.

---

### 🎭 Дзеркало

**Ефект:** Вибраний кластер "дублюється" — його дзеркальна копія з'являється на протилежному боці сітки.

- Копія має ті самі геноми але **переставлені у зворотному порядку** → мелодія інвертована / у ракоходному русі
- Fitness копії = 50 (нейтральний старт)

**Музичний ефект:** CanonicFormula — тема та її дзеркальне відображення звучать одночасно.

## 6. Петля гри та прогресія

### 6.1 Базова петля

```
ПОСІВ (Seeding phase)
  ├── Гравець вибирає стартовий патерн Life
  ├── Або завантажує збережений стан
  └── Встановлює початкові тональні регіони

        ↓

ЕВОЛЮЦІЯ (Evolution phase)
  ├── Симуляція іде автоматично
  ├── Гравець слухає мелодії кластерів
  ├── Оцінює кластери (подобається / ні)
  ├── Використовує інструменти середовища
  └── Чекає на події-каталізатори

        ↓

КРИСТАЛІЗАЦІЯ (Crystallization phase)
  ├── Деякі кластери досягають стабільного стану
  ├── Їх мелодії "застигають" у фінальній формі
  ├── Гравець зберігає вподобані
  └── Може продовжити або завершити сесію

        ↓

БІБЛІОТЕКА (Library phase)
  ├── Переглянути зібрані мелодії
  ├── Скомпонувати з них трек
  └── Експортувати як MIDI або аудіо
```

### 6.2 Стани рівноваги і що вони означають музично

| Стан Life | Поведінка мелодії |
|---|---|
| **Still life** (нерухомий) | Остинато: та сама фраза повторюється нескінченно |
| **Осцилятор** (2-period) | Два акорди що чергуються, як гойдалка |
| **Осцилятор** (higher period) | Складніший ритмічний паттерн |
| **Глайдер** | Мелодія "мандрує" через тональні регіони, модулюючи |
| **Глайдер-гармата** | Безперервне породження нових варіацій теми |
| **Хаос** | Атональний шум — сигнал для втручання гравця |

### 6.3 Умови "перемоги"

Гра не має традиційного кінця. Натомість — **досягнення**:

- 🎵 **Перший мотив** — зберегти першу мелодію що тривала > 20 тіків
- 🌿 **Садівник** — мати одночасно 5 кластерів з fitness > 70
- 🧬 **Спадщина** — простежити лінію ДНК від першої клітини до 10-го покоління
- 🌊 **Виживання хаосу** — зберегти кластер після події "Хвиля мутацій"
- 🎼 **Симфонія** — скомпонувати трек з 4+ різних збережених мелодій

### 7.1 Поділ відповідальностей

Уся гра реалізована на **чистому GDScript** в Godot 4.6 без зовнішніх залежностей. Зовнішнього сервера немає — усі підсистеми працюють у одному процесі.

### 7.2 Реалізований стек

| Компонент | Технологія |
|---|---|
| Симуляція Life | GDScript (life_grid.gd) |
| Кластеризація | BFS (cluster_manager.gd), не Union-Find |
| Аудіо движок | Чистий GDScript: AudioStreamGenerator, sine-wave additive synthesis |
| Схожість мелодій | Levenshtein distance (fitness_manager.gd) |
| Збереження стану | JSON snapshots кожні 5 тіків (ring buffer 10) |
| UI | Програмний GDScript (game_ui.gd), без .tscn |
| Рендеринг | OpenGL Compatibility (D3D12 на Windows) |

### 7.3 Сигнальна архітектура

Компоненти з'єднані через **Godot signals** у `main.gd`:
- `tick_completed` → ClusterManager → FitnessManager → CatalystEvents → GameUI
- `cell_liked` / `cell_muted` → FitnessManager
- `tool_changed` → GameUI (підсвітка кнопки)
- `event_requested` → CatalystEvents
- `seed_requested` / `clear_requested` → LifeGrid
